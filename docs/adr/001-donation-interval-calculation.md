# ADR-001: 献血間隔制限の計算アルゴリズム

## ステータス
Accepted

## コンテキスト
日本赤十字社の献血間隔ルールに従い、次回献血可能日を自動計算する必要がある。
ルールは献血種別と性別の組み合わせで決まり、さらに年間回数上限もある。

## 決定

### 間隔制限ルール
| 種別 | 男性 | 女性 |
|------|------|------|
| 全血400ml | 12週 | 16週 |
| 全血200ml | 4週 | 4週 |
| 成分献血 | 2週 | 2週 |

### 年間回数上限（全血のみ）
| 性別 | 上限 |
|------|------|
| 男性 | 3回/年 |
| 女性 | 2回/年 |

### 計算アルゴリズム
1. 献血記録を日付降順でソート
2. 最新の献血記録から間隔制限を計算（`lastDonatedAt + intervalWeeks * 7日`）
3. 過去1年間の全血献血回数をカウント
4. 年間上限に達している場合、最も古い年内全血献血の1年後を計算
5. 間隔制限日と年間上限日の**遅い方**を次回献血可能日とする
6. 次回献血可能日が今日以前なら「本日から献血可能」

### 「年」の定義
- 固定年度（4月〜3月）ではなく、**直近365日のスライディングウィンドウ**を採用
- 理由: 日本赤十字社の実運用に準拠、実装がシンプル

## 影響
- `DonationCalculator.NextAvailableDate()` が上記ロジックを実装
- テストは性別×献血種別の全組み合わせ + 年間上限 + 年末年始跨ぎをカバー
- 将来の献血種別追加時は `intervalWeeks()` にcase追加で対応可能
